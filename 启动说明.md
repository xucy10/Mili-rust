# Mili-rust 服务器启动指南

## 快速开始

### 方式一：双击启动（推荐）

1. 双击 `启动服务器.bat`
2. 等待编译完成（首次约5-10分钟）
3. 打开 Minecraft 1.20.1，连接 `localhost`

### 方式二：使用菜单启动

1. 双击 `run.bat`
2. 选择要运行的示例
3. 按提示操作

### 方式三：使用 PowerShell

```powershell
.\server.ps1
```

### 方式四：命令行

```bash
# 运行原版演示（红石/漏斗/作物）
cargo run --release -- example vanilla_demo

# 运行其他示例
cargo run --release -- example building
cargo run --release -- example parkour
cargo run --release -- example terrain
```

## 前置条件

### 必须安装

1. **Rust** - https://rustup.rs
2. **Visual Studio Build Tools** - https://visualstudio.microsoft.com/visual-cpp-build-tools/
   - 安装时选择 "C++ 桌面开发"

### 可选

- **Git** - 用于版本控制
- **VS Code** - 推荐的代码编辑器

## 可用示例

| 示例 | 说明 |
|------|------|
| vanilla_demo | 原版机制演示（红石、漏斗、作物、村民） |
| building | 方块放置和破坏 |
| parkour | 跑酷地图 |
| terrain | 地形生成 |
| game_of_life | 生命游戏 |
| anvil_loading | 加载 Minecraft 世界存档 |
| custom_npc | NPC对话系统 |
| chest | 箱子交互 |
| combat | 战斗系统 |

## 常见问题

### 编译失败

```
error: linker `link.exe` not found
```

**解决**: 安装 Visual Studio Build Tools

### 连接失败

```
连接被拒绝
```

**解决**: 确保服务器已启动，Minecraft版本为1.20.1

### 编译慢

首次编译较慢，后续会使用缓存。使用 `--release` 参数获得更好性能。

## 开发

### 添加新依赖

编辑 `Cargo.toml` 文件

### 运行测试

```bash
cargo test --workspace
```

### 编译检查

```bash
cargo check --workspace
```
